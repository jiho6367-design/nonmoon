# Nonmoon - 논문 관리 및 협업 플랫폼

## 📋 프로젝트 개요
Nonmoon은 **팀 기반의 논문 관리 시스템**입니다. 사용자는 논문을 업로드하고, PDF를 열람하며, 인용 카드를 생성·관리하고, AI 기반으로 요약할 수 있습니다. 팀원들과 함께 프로젝트를 구성하여 효율적으로 논문을 공동 관리합니다.

## 👥 팀 구성 및 역할


### 강지호 (팀장 2021145002 )
**백엔드 통합 및 메모카드 기능 구현**
- 팀원들의 코드 통합 및 보강
- **인용 카드(Citation Card)** 전체 기능
  - 표 뷰에서 인라인 수정 (✏️ 클릭 → 제목/저자/연도/인용문 직접 수정)
  - PATCH 요청으로 서버 반영 및 행 즉시 갱신
  - 행 단위 삭제 처리
  
- **고급 필터링 & 검색**
  - 본문/주제 검색 + 저자 + 연도 + 북마크 조합 필터링
  - 검색어 없이 저자·연도만으로도 검색 가능
  - 최신순/과거순 정렬
  
- **논문 요약 관리**
  - 모든 요약을 전역으로 조회·수정·삭제 (논문/계정 무관)
  - 요약 저장 시 관련 논문 메타·파일명·스토리지 파일명 함께 보존
  
- **요약-PDF 동기화**
  - 요약 선택 → 좌측 PDF 뷰어가 해당 요약의 PDF로 교체
  - 상단 파일명 동기화 (pdfName/storedFileName)
  - 요약 패널 닫기 → 원래 논문 PDF로 복귀
  
- **업로더/작성자 정보 유지**
  - 카드·요약 업로드 시 uploader/author 필드 그대로 저장·표시

### 박예슬 (2023145036)
**논문 페이지 구조 및 프론트엔드 UI 담당 + 버그 수정 및 프로젝트 통합**
- **PaperListPage**: 논문 전체 목록 페이지
- **PaperDetailPage**: 논문 상세 페이지 (PDF 뷰어 + 인용 카드)
- **PaperCard**: 논문 카드 컴포넌트
- **PaperUploadModal**: 논문 업로드 모달

- **논문 상세 페이지**
  - PDF 뷰어와 인용 카드를 한 화면에 배치하는 레이아웃 구조 설계
  - 인용 카드 선택 시 PDF 페이지 동기화 로직 구현
  
- **논문 업로드** 
  - 파일 선택 및 메타정보(제목/저자/연도) 입력 폼 구현
  - 업로드 컴포넌트트 생성, 수정, 삭제 기능 구현
    
- **버그 수정 및 프로젝트 통합**
  - 파일 구조 개선 - 두 개의 분리된 프로젝트(user-auth, client-react) 통합
  - 버전 호환성 해결 - 라이브러리/코드 버전 맞춤
  - 팀 관리 → 논문 페이지 네비게이션 버그 수정 - 페이지 이동 안 되던 거 해결

## 🛠 기술 스택

### Frontend
- **React** 18.2.0
- **PDF 뷰어**: react-pdf + pdfjs-dist
- **세션 관리**: express-session

### Backend
- **Node.js + Express** 5.1.0 / 4.18.2
- **데이터베이스**: MySQL 2
- **파일 업로드**: Multer
- **PDF 파싱**: pdf-parse
- **보안**: bcrypt (비밀번호 암호화), CORS

## 🚀 주요 기능

| 기능 | 설명 |
|------|------|
| **로그인/회원가입** | 사용자 계정 생성 및 인증 |
| **팀 프로젝트 관리** | 팀 생성, 팀원 추가, 권한 관리 |
| **논문 관리** | 논문 업로드, 논문 시각화, 메타정보(제목/저자/연도) 수정, 팀원과 공유|
| **인용 카드** | 논문별 인용 정보 관리 (생성, 수정, 삭제, 검색, 필터링) |

### 3. 요약-PDF 동기화
사용자가 요약을 선택하면 좌측 PDF 뷰어가 해당 요약의 PDF로 자동 교체되고, 요약 패널을 닫으면 원래 논문으로 복귀합니다.

### 4. 메타정보 보존
논문 업로드 시 업로더 이름, 요약 작성 시 작성자 이름을 함께 저장하여 출처를 명확히 합니다.
